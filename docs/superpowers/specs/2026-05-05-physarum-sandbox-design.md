# 黏菌沙盒设计文档

## 一、项目概述

这是一个基于浏览器的黏菌沙盒网站，核心聚焦在“饲养与模拟”，而不是科普介绍。用户通过在二维培养皿中布置食物、有害物质和障碍，观察一个类似 `Physarum polycephalum` 的数字黏菌自行扩张、连接、收缩、休眠与复苏。

首版目标不是严格复刻生物实验，而是在视觉和行为上尽量接近真实黏菌，让用户能够快速搭建一个场景，并稳定看到这些有趣结果：

- 朝营养源扩张
- 绕开有害区域和障碍
- 形成主脉络并加粗
- 回收低价值支路
- 在不利条件下菌核化
- 条件恢复后再次苏醒

## 二、产品方向

### 2.1 已确定的产品形态

- 产品类型：互动养成 / 模拟网站
- 体验类型：沙盒
- 首版不做科普内容
- 首版不做任务关卡
- 首版不模拟温度、湿度等环境参数

### 2.2 核心体验

产品应更像一个“可观察生态箱”，而不是一个高频点击的宠物游戏。黏菌自己演化，用户负责布置场景与调节观察速度。

核心循环如下：

1. 放置菌核或初始接种点
2. 放置不同类型的食物
3. 放置有害物质和障碍
4. 调整模拟速度
5. 观察扩张、寻路、收缩、休眠和复苏
6. 重置场景并继续实验

## 三、视觉方向

整体视觉应偏“接近真实黏菌”的有机生命感，而不是科学图表风格，也不是像素游戏风格。

视觉重点如下：

- 边缘轮廓要有机，不应过于硬朗
- 颜色变化用于表达状态变化
- 连接成功后主脉络应明显加粗
- 动态效果应缓慢、湿润、像活体
- 食物、障碍、有害区域、菌核状态要有清晰区分

## 四、首版功能范围

首版只围绕模拟沙盒展开，避免加入分散注意力的系统。

### 4.1 核心模块

1. 主培养皿画布
   - 显示黏菌本体、脉络、前沿、食物、有害物和障碍

2. 投放工具
   - 菌核
   - 食物
   - 有害物质
   - 障碍
   - 擦除 / 清除笔刷

3. 时间控制
   - 暂停
   - 1x
   - 5x
   - 20x

4. 状态观察面板
   - 占据面积
   - 活跃前沿数量
   - 已连接食物数量
   - 菌核数量
   - 整体活性趋势

5. 重置工具
   - 一键清空
   - 随机场景
   - 重新播种

6. 导出
   - 首版只做静态截图导出

### 4.2 明确不做的内容

- 账号系统和云端存档
- 社区分享
- 任务、目标或闯关系统
- 科普百科内容
- 温度、湿度等完整环境模拟
- 严格生化级别的真实性
- 多菌落竞争
- 高级学习 / 习惯化系统

## 五、模拟方案总览

### 5.1 推荐的模拟抽象

首版建议使用以下混合模型：

- 场驱动环境
- 前沿生长
- 脉络强化与回收
- 菌核状态切换

这条路线比“直接计算最短路”更像真实黏菌的探索行为，也比“全流体 / 全粒子模拟”更容易控制和落地，适合作为第一版实现方向。

### 5.2 世界表示

培养皿内部至少维护以下六层状态：

1. 营养吸引场
   - 由食物持续释放
   - 向周围扩散并随距离和时间衰减

2. 毒害 / 排斥场
   - 由有害物质释放
   - 不同类型可以选择扩散或局部固定

3. 障碍场
   - 硬障碍完全不可通过
   - 软障碍会提高推进成本，鼓励黏菌绕行或贴边移动

4. 黏菌活性场
   - 表示该区域是活跃前沿、稳定脉络、衰退组织还是菌核

5. 脉络厚度场
   - 表示运输通道的重要程度与稳定程度

6. 局部资源 / 流量场
   - 用于判断某条路线是否值得继续保留

## 六、黏菌状态设计

黏菌本体建议使用四种主要状态。

### 6.1 活跃前沿

- 颜色最亮
- 负责向有利方向推进
- 是主要生长边缘

### 6.2 稳定脉络

- 颜色略深，宽度更粗
- 负责维持食物与菌核之间的有效连接

### 6.3 衰退组织

- 颜色变暗，厚度变薄
- 表示该区域正在失去价值或受到损伤

### 6.4 菌核

- 偏褐黄或橙褐色
- 静止不动
- 可在条件改善后重新激活

## 七、单步演化流程

每个模拟 tick 建议按以下顺序更新：

1. 更新环境场
   - 营养场扩散并衰减
   - 毒害场扩散并衰减

2. 感知邻域
   - 活跃前沿读取周围吸引、排斥和阻力信息
   - 优先朝高收益、低毒害、低阻力方向推进
   - 保留一定随机性，避免行为过于机械

3. 推进前沿
   - 占领新的区域
   - 在新位置沉积黏菌质量

4. 强化有效通路
   - 成功连接到高价值食物的路径会逐步增厚并稳定下来

5. 回收低价值支路
   - 低流量、低收益、远离有效连接的区域开始变薄和收缩

6. 结算伤害
   - 位于有害区域中的组织会被减速、压制、收缩或击穿

7. 判定菌核化
   - 长期饥饿、隔离或受损的组织进入菌核态
   - 菌核附近若再次出现足够强的营养梯度，则可恢复活性

## 八、物质系统设计

物质系统应当足以制造差异化行为，但不要复杂到难以调参。

### 8.1 食物类型

首版建议保留 3 类食物。

#### 1. 稳定食物

- 吸引力中高
- 营养容量高
- 容易形成稳定主脉络

#### 2. 速效食物

- 短时间吸引力很强
- 持续时间较短
- 容易引发快速扩张，但很快耗尽

#### 3. 扩散食物

- 单点吸引力较弱
- 影响范围较大
- 适合远距离引导和慢速诱导

每类食物统一使用以下参数：

- `attraction_strength`
- `range`
- `nutrition_capacity`
- `decay_rate`

### 8.2 有害物质类型

有害物不应统一表现为“碰到就死”，而应在“排斥、压制、损伤、灭活”之间形成差异。

首版建议保留 4 类有害物。

#### 1. 盐

- 高排斥
- 中等伤害
- 适合做边界、禁区和围栏

#### 2. 咖啡因 / 奎宁类排斥物

- 高排斥
- 低到中等伤害
- 更适合诱导绕行，而不是直接杀伤

#### 3. 强光斑块

- 中等排斥
- 中等伤害
- 作为环境压力区存在，而不是普通障碍

#### 4. 消毒剂 / 重污染区

- 中等排斥
- 高伤害
- 可以局部灭活黏菌组织

每类有害物统一使用以下参数：

- `repulsion_strength`
- `toxicity`
- `range`
- `decay_rate`

### 8.3 障碍类型

首版建议保留两类障碍。

#### 1. 硬障碍

- 完全不可通行
- 适合墙、隔板、石块等结构

#### 2. 软障碍

- 可以通过，但成本更高
- 更适合制造“贴边绕行”的行为

## 九、菌核设计

菌核不应只是开局放置点，而应是一个完整的生命状态。

### 9.1 菌核化触发条件

当局部组织长期处于以下情况时，可转入菌核：

- 周围长期缺乏可达营养
- 与高价值运输通路断开
- 重复遭受有害物伤害
- 局部流量长期过低

### 9.2 菌核复苏条件

菌核可在以下条件下重新激活：

- 附近出现足够强的营养梯度
- 相邻活体组织重新连接到它

这样系统会表现出“死亡与复苏”的生命感，而不是简单删除。

## 十、关键行为目标

如果模拟方向正确，首版应稳定产生以下现象：

1. 黏菌朝最近或最强食物源扩张
2. 遇到盐、强光或排斥物时明显绕开
3. 同时面对多个食物时，先广泛分叉再逐步收敛
4. 成功连接后主脉络明显加粗
5. 低价值支路逐渐变暗、变细、收缩
6. 被隔离或长期受损的区域转入菌核态
7. 新食物出现在菌核附近时，黏菌重新复活

如果这 7 类结果都能稳定出现，说明首版模拟方向是成功的。

## 十一、最重要的行为原则

### 11.1 必须保留的特征

1. 扩张速度应快于回收速度
   - 这样才有“先探索、后整理”的观感

2. 局部决策生成全局结构
   - 不应直接计算完美最短路

3. 状态切换应有惯性
   - 接触、连接、增厚、衰退、菌核化都应逐步发生

4. 有害物应先表现为压力，再表现为致命性
   - 先回避、减速、萎缩，再进入严重损伤或休眠

### 11.2 应避免的结果

1. 一开始就直接走出最优路径
2. 像油漆一样均匀铺开，没有脉络回收
3. 所有有害物都表现为一碰就死
4. 所有食物除了颜色外没有行为差异

## 十二、统一参数结构

为了便于调参与扩展，推荐所有物质使用统一参数结构。

### 12.1 食物参数

- `attraction_strength`
- `range`
- `nutrition_capacity`
- `decay_rate`

### 12.2 有害物参数

- `repulsion_strength`
- `toxicity`
- `range`
- `decay_rate`

### 12.3 黏菌组织参数

- `mass`
- `activity`
- `thickness`
- `damage`
- `dormancy`

## 十三、时间系统

时间系统应支持不同观察速度，但不改变规则本身。

- `1x`：适合观察边缘变化和状态切换
- `5x`：适合作为默认观看速度
- `20x`：适合观察长期结构变化和整体兴衰

不同速度模式下，应保持同一套演化规则，只调整 tick 推进频率或每次推进的步数。

## 十四、推荐技术路线

从产品目标与首版工程可控性来看，推荐采用以下实现方向：

- 二维培养皿模拟
- 标量场表示吸引、排斥和占据状态
- 每个 tick 更新活跃前沿
- 对已连接路径做强化，对低价值路径做回收
- 用简单状态机处理菌核化与复苏

这条路线在真实性、可控性、性能和实现难度之间最平衡。

## 十五、主要风险与应对

### 15.1 主要风险

1. 生长规则过于简单，结果会像颜料扩散而不是黏菌
2. 模拟机制过于复杂，首版难以调通
3. 物质种类过多，导致平衡混乱
4. 控制面板过重，削弱沙盒的即时性

### 15.2 应对方式

1. 优先把前沿行为与脉络回收做好
2. 所有物质统一参数结构，方便整体调参
3. 首版控制在 3 类食物和 4 类有害物
4. 尽量让颜色和厚度承担状态表达，减少说明负担

## 十六、参考结论

本设计的抽象方向主要依据以下文献中反复出现的行为特征：

- 黏菌会沿营养梯度扩张
- 它会先广泛探索，再整理运输网络
- 内部脉动和延迟结构调整是重要特征
- 食物质量会影响搜索策略
- 有害刺激通常先表现为回避、减速、损伤或休眠，而不是瞬时死亡
